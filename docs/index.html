features

Built for security. Made for the terminal.

🔒
TLS 1.3 transport
Every connection uses AES-256-GCM or ChaCha20. No weak ciphers. SSLv2/3 and TLS 1.0/1.1 are disabled.
TLS 1.3
🔐
Optional E2E layer
Add a second AES-256-GCM encryption layer on top of TLS with --e2e. Double-wrapped by default.
AES-256-GCM
💬
Two-way live chat
Full-duplex real-time chat with the chat command. Both sides send and receive simultaneously.
full-duplex
🧹
Zero persistence
Messages are printed once and never written to disk. No logs. No database. Nothing left behind.
stateless
⚡
Fire-and-forget
Send a single message and disconnect with snd. Receive and print with rcv. Simple and composable.
lightweight
🐧
Linux native
Systemd service unit included. Run rcv as a background daemon on any Arch, Debian, or Ubuntu system.
systemd
installation

Up in 5 minutes

Requires Python 3.11+ and OpenSSL (pre-installed on most Linux systems).

bash
# 1. clone $ git clone https://github.com/danshu3007-lang/secure-messenger $ cd secure-messenger # 2. install $ python3 -m venv .venv && source .venv/bin/activate $ pip install . # 3. generate test certificates $ bash tests/certs/gen_test_certs.sh
bash — test on one machine (2 terminals)
# terminal 1 $ MESSENGER_CERT_DIR=tests/certs chat 127.0.0.1 -lp 8443 -pp 8444 # terminal 2 $ MESSENGER_CERT_DIR=tests/certs chat 127.0.0.1 -lp 8444 -pp 8443
security model

What is and isn’t protected

property implementation status
Confidentiality TLS 1.3 — AES-256-GCM or ChaCha20-Poly1305 protected
Integrity GCM authentication tag — tampering is detected and rejected protected
Forward secrecy TLS 1.3 ephemeral key exchange per session protected
Message storage Messages printed once — never written to disk zero persistence
IP / metadata IP addresses, connection timing, and packet sizes are visible on the network not protected
