# Architecture Overview

## Design Philosophy

`secure-messenger` is intentionally simple and stateless. There is no server process, no database, no user registry, and no message queue. Every connection is point-to-point, ephemeral, and encrypted.

## Encryption Stack

```
┌─────────────────────────────────────┐
│         Application Message         │
└──────────────────┬──────────────────┘
                   │  (optional)
                   ▼
┌─────────────────────────────────────┐
│   E2E Layer: AES-256-GCM            │
│   Pre-shared key, exchanged OOB     │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│   TLS 1.3 Transport                 │
│   AES-256-GCM or ChaCha20-Poly1305  │
│   Ephemeral key exchange            │
│   Certificate authentication        │
└──────────────────┬──────────────────┘
                   │
                   ▼
               TCP Socket
```

## chat Command — How Full-Duplex Works

`chat` opens **two separate TLS connections** simultaneously:

- One connection where you are the **server** (listening on `-lp`)
- One connection where you are the **client** (connecting to peer's `-pp`)

Two threads handle sending and receiving independently. This avoids the complexity of half-duplex turn-taking while keeping the code straightforward.

## Certificate Model

For testing: self-signed certs generated by `tests/certs/gen_test_certs.sh`

For LAN use:
1. One machine acts as the informal CA using `certs/gen_certs.sh`
2. The CA's `ca.crt` is copied to the other machine (via USB or SCP)
3. Both machines trust the same CA — TLS handshake succeeds

Optional mTLS: the `--mtls` flag on `rcv` requires the connecting client to also present a valid certificate signed by the same CA.

---

## Non-Goals

- **No relay server** — messages go directly machine-to-machine
- **No accounts or auth** — certificate trust is the only identity mechanism
- **No persistence** — there is no log, database, or history
- **No GUI** — terminal only by design
- **No WAN support** — designed for trusted LAN environments
